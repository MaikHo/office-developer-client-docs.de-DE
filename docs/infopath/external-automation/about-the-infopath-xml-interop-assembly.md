# <a name="about-the-infopath-xml-interop-assembly"></a>Informationen zur InfoPath-XML-Interop-Assembly

Die InfoPath-XML-Interop-Assembly wird bereitgestellt, um die Interoperabilität zwischen verwaltetem Code und dem von Microsoft XML Core Services (MSXML) verfügbar gemachten COM-Server von externen Anwendungen, die InfoPath automatisieren, zu ermöglichen.

Die Option **.NET-Programmierunterstützung** im InfoPath-Setupprogramm installiert drei Interop-Assemblys. Interop-Assemblys sind .NET-Assemblys, die als Verbindung zwischen verwaltetem und nicht verwaltetem Code dienen, indem sie COM-Objektmember entsprechenden verwalteten .NET-Membern zuordnen. Eine dieser Assemblys, Microsoft. Office. Interop. InfoPath. Xml. dll, stellt die Member des [Microsoft. Office. Interop. InfoPath. XML-](https://docs.microsoft.com/dotnet/api/microsoft.office.interop.infopath.xml?view=infopath-external) Namespaces bereit, der für die Arbeit mit vom com-Server für Microsoft XML Core Services (MSXML) verfügbar gemachten Membern verwendet wird. von externen Anwendungen, die InfoPath mithilfe von verwaltetem Code automatisieren. 
  
> [!NOTE]
> Die Verweise auf die Interop-Assemblys "Microsoft. Office. Interop. InfoPath. dll" und "Microsoft. Office. Interop. InfoPath. Xml. dll" müssen manuell erstellt werden. Weitere Informationen zur externen Automatisierung finden Sie unter [Szenarios und Beispiele für externe Automatisierung](external-automation-scenarios-and-examples.md).
