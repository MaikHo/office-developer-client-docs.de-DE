# <a name="errors"></a>Fehler

**Betrifft**: Access 2013, Office 2013

Jeder Vorgang, an dem ADO-Objekte beteiligt sind, kann mindestens einen Anbieterfehler generieren. Sobald ein Fehler auftritt, wird mindestens ein **Error** -Objekt in der **Errors** -Auflistung des **Connection** -Objekts eingefügt. Ausführliche Informationen zum Behandeln von Warnungen und Fehler in der ADO-Anwendung finden Sie unter [Kapitel 6: Fehlerbehandlung](chapter-6-error-handling.md).

Anwendungsfehler können von einem separaten Mechanismus ausgelöst werden. Beispielsweise enthält das **Err** -Objekt in Visual Basic Fehler auf Anwendungsebene.
