# <a name="testing-and-debugging"></a>Testen und Debuggen

  
  
**Betrifft**: Outlook 
  
Teststrategien unterscheiden sich je nachdem, ob Sie einen Client oder Dienstanbieter entwickeln. Da eine Clientanwendung eine oder mehrere Dienstanbieter Betrieb erforderlich sind, sollten Clients in einer Umgebung mit unterschiedlichen Dienstanbieter getestet werden.
  
Dienstanbieter, sollten jedoch vor der Integration mit anderen Anbietern isoliert getestet werden. MAPI bietet Tools, die zum Testen der Features von einem Dienstanbieter eines bestimmten Typs vorgesehen sind. Die beispielanwendung [MFCMAPI (engl.)](http://go.microsoft.com/fwlink/?LinkId=124154) zeigt, wie die Features des Adressbuch-Dienstanbieter testen und Speicheranbieter Works mit einer Nachricht.
