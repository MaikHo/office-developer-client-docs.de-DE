<span data-ttu-id="afb39-107">Ein Client kann einen threadsicheren Benachrichtigungsrückruf implementieren, indem er das MAPI-Dienstprogramm [HrThisThreadAdviseSink](hrthisthreadadvisesink.md).</span><span class="sxs-lookup"><span data-stu-id="afb39-107">A client can implement a thread-safe notification callback by calling the MAPI utility [HrThisThreadAdviseSink](hrthisthreadadvisesink.md).</span></span> <span data-ttu-id="afb39-108">**HrThisThreadAdviseSink** transformiert eine nicht threadsichere Advise-Senke in ein threadsicheres.</span><span class="sxs-lookup"><span data-stu-id="afb39-108">**HrThisThreadAdviseSink** transforms a non-thread-safe advise sink into a thread-safe one.</span></span> <span data-ttu-id="afb39-109">Für Status Rückrufe gibt es kein solches Hilfsprogramm.</span><span class="sxs-lookup"><span data-stu-id="afb39-109">For progress callbacks, there is no such utility.</span></span> <span data-ttu-id="afb39-110">Ein Client kann das MAPI-threadsichere Progress-Objekt verwenden oder manuell erstellen.</span><span class="sxs-lookup"><span data-stu-id="afb39-110">A client can choose to use the MAPI thread-safe progress object or create one manually.</span></span> 
  
<span data-ttu-id="afb39-111">Ein threadsicheres Objekt ist möglicherweise auch nicht Thread fähig.</span><span class="sxs-lookup"><span data-stu-id="afb39-111">A thread-safe object might or might not also be thread-aware.</span></span> <span data-ttu-id="afb39-112">Ein Thread fähiges Objekt verwaltet einen separaten Kontext für jeden Thread, der es verwendet.</span><span class="sxs-lookup"><span data-stu-id="afb39-112">A thread-aware object maintains a separate context for every thread that is using it.</span></span> <span data-ttu-id="afb39-113">Dienstanbieter müssen die Thread Erkennung in ihren threadsicheren Objekten nicht unterstützen, obwohl die Unterstützung der Thread Erkennung in einigen Situationen nützlich sein kann.</span><span class="sxs-lookup"><span data-stu-id="afb39-113">Service providers are not required to support thread-awareness in their thread-safe objects, although supporting thread-awareness can be useful in some situations.</span></span> <span data-ttu-id="afb39-114">Zwei MAPI-Tabellen stellen immer ihren eigenen Kontext per Definition bereit.</span><span class="sxs-lookup"><span data-stu-id="afb39-114">Two MAPI tables always provide their own context by definition.</span></span> <span data-ttu-id="afb39-115">Eine Tabelle, die in verschiedenen Threads verwendet wird, bietet keinen eindeutigen Kontext.</span><span class="sxs-lookup"><span data-stu-id="afb39-115">One table used on different threads does not and should not provide unique context.</span></span>
  
<span data-ttu-id="afb39-116">Ein Client kann zwischen dem Empfang von Benachrichtigungen für den gleichen Thread, der für den **MAPIInitialize** -Aufruf verwendet wurde, im gleichen Thread, der für den **Advise** -Aufruf verwendet wurde, oder in einem separaten Thread, der im Besitz von MAPI ist, wählen.</span><span class="sxs-lookup"><span data-stu-id="afb39-116">A client can choose between receiving notifications on the same thread that was used for the **MAPIInitialize** call, on the same thread that was used for the **Advise** call, or on a separate thread owned by MAPI.</span></span> <span data-ttu-id="afb39-117">Um sicherzustellen, dass Benachrichtigungen für den gleichen Thread eingehen, der zum Aufrufen von **MAPIInitialize**verwendet wurde, Ruft ein Client [MAPIInitialize](mapiinitialize.md) auf und übergibt 0 (null) im **ulFlags** -Element der [MAPIINIT_0](mapiinit_0.md) -Struktur.</span><span class="sxs-lookup"><span data-stu-id="afb39-117">To ensure that notifications arrive on the same thread that was used to call **MAPIInitialize**, a client calls [MAPIInitialize](mapiinitialize.md) and passes zero in the **ulFlags** member of the [MAPIINIT_0](mapiinit_0.md) structure.</span></span> <span data-ttu-id="afb39-118">Benachrichtigungen werden dann während der Hauptmeldungsschleife übermittelt.</span><span class="sxs-lookup"><span data-stu-id="afb39-118">Notifications are then delivered during the main message loop.</span></span> 
  
<span data-ttu-id="afb39-119">Um Benachrichtigungen über den MAPI-besessenen Thread zu erhalten, Ruft ein Client **MAPIInitialize** mit dem **ulFlags** -Element der **MAPIINIT_0** -Struktur auf MAPI_MULTITHREAD_NOTIFICATIONS festgelegt.</span><span class="sxs-lookup"><span data-stu-id="afb39-119">To receive notifications on the MAPI-owned thread, a client calls **MAPIInitialize** with the **ulFlags** member of the **MAPIINIT_0** structure set to MAPI_MULTITHREAD_NOTIFICATIONS.</span></span> <span data-ttu-id="afb39-120">Der **Advise** -Aufruf erfolgt mit dem Advise-Objekt des Clients und nicht mit einer umbrochenen Version.</span><span class="sxs-lookup"><span data-stu-id="afb39-120">The **Advise** call is made with the client's advise sink object rather than a wrapped version.</span></span> 
  
<span data-ttu-id="afb39-121">Um sicherzustellen, dass Benachrichtigungen für den gleichen Thread eingehen, der zum Aufrufen von **Advise**verwendet wurde, Ruft ein Client [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) auf und übergibt die \*\*\*\* neu erstellte umschlossene Advise-Senke anstatt der ursprünglichen Advise-Senke.</span><span class="sxs-lookup"><span data-stu-id="afb39-121">To ensure that notifications arrive on the same thread that was used to call **Advise**, a client calls [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) and passes the newly created wrapped advise sink to **Advise** rather than the original advise sink.</span></span> <span data-ttu-id="afb39-122">**MAPIInitialize** kann mit einem der beiden Flags-Werte aufgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="afb39-122">**MAPIInitialize** can be called with either flag value.</span></span>
