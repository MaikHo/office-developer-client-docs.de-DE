# <a name="idle-state"></a>Status „Leerlauf“

  
  
**Betrifft**: Outlook 2013 | Outlook 2016 
  
 In diesem Thema wird beschrieben, was geschieht, während die Leerlauf des Computers Zustand Replikation. 
  
## <a name="quick-info"></a>QuickInfo

|||
|:-----|:-----|
|State-ID:  <br/> |**LR_SYNC_IDLE** <br/> |
|Verwandte-Datenstruktur:  <br/> | *None*  <br/> |
|Aus diesem Zustand:  <br/> | *Nicht zutreffend*  <br/> |
|Diesen Status:  <br/> |[Synchronisieren von Zustand](synchronize-state.md) <br/> |
   
> [!NOTE]
> Das Zustandsautomat Replikation ist ein deterministisch Zustandsautomat. Ein Client, der von einem Zustand zu einem anderen Unternehmen muss schließlich auf die frühere letztere zurückgeben. 
  
## <a name="description"></a>Beschreibung

In diesem Status ohne Wirkung. Ein lokales Speichers wird in diesem Status vor Replikation ausgeführt werden und nach dem Replikation abgeschlossen ist.
