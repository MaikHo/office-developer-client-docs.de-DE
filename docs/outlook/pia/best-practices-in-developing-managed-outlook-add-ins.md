# <a name="best-practices-in-developing-managed-outlook-add-ins"></a>Bewährte Methoden beim Entwickeln von verwalteten Outlook-Add-Ins

Auch wenn Sie mit den Outlook-Interopassemblys verwaltete Lösungen schreiben können, die mit Outlook zusammenarbeiten, wurde Outlook bereits vor der Einführung von .NET Framework entwickelt, um die Programmierung mit nicht verwalteten Sprachen wie Visual Basic for Applications (VBA) und Visual Basic zu unterstützen. In diesem Thema sind die wichtigsten Gesichtspunkte aufgeführt, die Sie beim Entwickeln eines verwalteten Add-Ins für Outlook beachten sollten.

- [Systematisches Freigeben von Objekten](systematically-releasing-objects.md)
- [Verwenden einer separaten Anwendungsdomäne zur Vermeidung von Abstürzen](using-a-separate-application-domain-to-avoid-crashing.md)
- [Verwenden eines vertrauenswürdigen, von den Office Developer Tools für Visual Studio bereitgestellten Anwendungsobjekts](using-a-trusted-application-object-provided-by-office-developer-tools-for-visual-studio.md)
- [Geeignetes Festlegen von Bereichen für Variablen in Ereignishandlern](scoping-variables-appropriately-in-event-handlers.md)
- [Vermeiden von nicht unterstützten Technologien in verwalteten Outlook-Add-Ins](avoiding-unsupported-technologies-in-managed-outlook-add-ins.md)
- [Verwenden von spätem Binden bei mehreren Version von Outlook](using-late-binding-if-depending-on-multiple-versions-of-outlook.md)
